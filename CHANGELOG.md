# Changelog

本项目变更记录。

## [v1.6] - 2026-04-11

- 修复外部 M3U 分组合并过度依赖完整 `group-title` 的问题；当上游把 `🔮[主用]港澳台直播` 调整为 `🔮港澳台直播` 之类的名称时，现可按关键词模糊命中并继续合并。
- 新增外部分组规范化匹配逻辑：支持规范化后相等、包含关系，以及不少于 3 个连续字符的公共片段匹配，兼容表情、标签和前缀波动。
- 调整 `EXTERNAL_GROUP_TITLES` 的映射行为，字典配置改为基于命中的目标关键词重命名，推荐直接使用 `"港澳台": "港澳台"` 这类稳定配置。
- 同步更新 README 外部 M3U 配置示例和发布文档。

## [v1.5] - 2026-03-27

**我们将在** **tv.py** **中引入 Python 标准库** **argparse**，允许你在运行命令时直接覆盖配置文件中的这几个核心开关。

**新增的命令行参数：**

* **--skip-epg** **或** **-se**: 临时禁用 EPG 下载（覆盖 **ENABLE_EPG_DOWNLOAD=False**）
* **--skip-check** **或** **-sc**: 临时禁用底层的 IPTV 检测引擎（覆盖 **ENABLE_STREAM_CHECK=False**）
* **--skip-probe** **或** **-sp**: 允许检测存活，但临时禁用极度耗时的 FFprobe 画质探测（覆盖 **ENABLE_PROBE=False**）

## [v1.4] - 2026-03-27

🌟 重磅更新：引入底层视频流智能检测引擎 (`iptv_checker_v3.py` 融合版)！

- **零磁盘损耗内存检测**：`tv.py` 提取完频道后，在内存中直接调用异步引擎进行存活检测。踢出失效源后再生成 M3U 文件，确保生成的列表 100% 纯净可播。
- **FFprobe 深度画质解析**：开启 `ENABLE_PROBE` 后，自动下载底层视频切片，精准识别频道真实画质，并在输出时自动为频道名称打上 `[1080P][H264]` 等高清标签。
- **三层极限防盗链对抗**：独创“直击底层切片 -> 回退解析 M3U8 -> 正则文本捡漏”三重机制，将高防盗链节点（如 Astro / TVB）的测活与画质解析成功率推向极限。
- **无感智能缓存系统**：检测数据自动记入 `config/probe_cache.json`。缓存有效期内每天高频重跑脚本只会执行极速测活，极大节省 NAS 性能和网络带宽。
- **日志系统规范化**：将检测引擎的 `checker_final.log` 和死链分析报告 `failed_channels.txt` 统一归档至 `log/` 目录下。

## [v1.3] - 2026-03-25

🚀 核心提速与防风控：引入 requests.Session 长连接复用，大幅减少 TCP/TLS 握手开销；新增 EPG_REQUEST_DELAY 和并发连接限制，平滑请求节奏，完美规避高频抓取导致的 403 拦截风控。
📉 极低内存优化：废弃 minidom 库，改用原生 xml.etree.ElementTree.indent 进行 XML 格式化，生成万级节目单时内存占用暴降 80% 以上，彻底杜绝小内存设备卡死问题。
⚡ 正则匹配加速：将黑名单标题列表预编译为底层正则表达式引擎，匹配过滤速度呈指数级提升。
🐛 修复排序覆盖Bug：重写了自定义排序引擎，修复了外部/自定义频道中“同名但播放源不同”的频道在字典处理时互相覆盖丢失的问题，现可完美保留所有同名备用源。
♻️ 代码重构：抽离 Nginx 代理地址统一替换逻辑，规范脚本运行生命周期，消除 M3U 生成函数中大量的冗余复制粘贴代码，大幅提升代码可读性与可二次开发性。

## [v1.2] - 2026-03-24

- 外部 M3U 缓存回退：新增 `CACHE_M3U_FILENAME` 配置，默认使用 `cache.m3u` 作为外部 M3U 本地缓存文件。
- 下载容错增强：外部 M3U 改为默认优先网络更新，网络请求失败时自动回退到本地缓存，避免合并结果丢失外部频道。
- 外部频道去重：外部 M3U 解析阶段新增按 URL 去重逻辑，遇到“同 URL 不同别名”时仅保留第一次出现的频道。
- 日志增强：`channel_processing.log` 增加外部 M3U 来源、缓存文件路径和 URL 重复过滤统计，便于排查外部合并结果。
- 文档同步：更新 README 外部 M3U 配置说明，并同步项目版本到 v1.2。

## [v1.1] - 2026-03-24

1- EPG 设置简化：新增 `EPG_DAY_OFFSETS: [9]` 简化写法，自动展开为包含明天的 9 天 EPG 下载窗口。

- 配置统一：`config.json` 和 `myconfig.json` 默认同步更新为简化后的 9 天配置。
- 版本更迭：版本号统一更新为 v1.1。

## [v1.0] - 2026-03-23

- 配置加载改进：`tv.py` 启动时按 `config/config.json` -> `config/myconfig.json` 覆盖加载配置。
- EPG 下载改进：新增并启用 `EPG_DAY_OFFSETS` 多天下载窗口（例如 `[-5,-4,-3,-2,-1,0,1]`）。
- EPG 合成修复：修正节目去重字段匹配，恢复多天场景下节目条目数量异常偏低的问题。
- 路径兼容修复：配置文件路径改为基于脚本目录解析，避免 UNC/CWD 变化导致读取失败。
- 日志路径修复：`channel_processing.log` 与 `epg_statistics.log` 固定输出到 `log/` 目录并自动创建目录。
- 文档更新：补充 Windows UNC 下稳定运行命令与配置使用说明。
- 目录精简：自定义 JSON 统一迁移到 `config/`，运行日志统一写入 `log/`。
- 配置外置：支持 `config/config.json` + `config/myconfig.json` 分层覆盖。
- Git 规则更新：远程仓库统一使用 SSH 方式上传到 GitHub。
